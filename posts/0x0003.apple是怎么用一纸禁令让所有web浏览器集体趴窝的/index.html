0x0003.Apple是怎么用一纸禁令让所有Web浏览器集体趴窝的？

七月 17, 2025

——论iOS浏览器生态的「伪竞争」与WebKit霸权#

阅读须知：本文带有强烈技术批判倾向，果粉、Safari工程师及Apple法务部人员请谨慎服用。

1. 表面繁荣：iOS上的「浏览器选择」#

打开App Store，搜索「浏览器」，你会看到：

  • Google Chrome
  • Microsoft Edge
  • Mozilla Firefox
  • Brave / Opera / DuckDuckGo…

看起来百花齐放？ 但真相是——它们全是Safari的换皮马甲。

2. Apple的「核捆绑」政策#

2.1 App Store的霸王条款#

根据Apple的App Store审核指南 2.5.6条：

“Apps that browse the web must use the appropriate WebKit framework and WebKit Javascript.”

翻译成人话：

「所有iOS浏览器必须用Apple的WebKit内核，禁止第三方引擎。」

这意味着：

  • Chrome on iOS ≠ 真正的Chrome（没有Blink/V8引擎）
  • Firefox on iOS ≠ 真正的Firefox（没有Gecko引擎）
  • 所有浏览器本质上只是Safari的皮肤+书签同步工具

2.2 技术验证：如何证明它们都是Safari？#

打开iOS的Chrome/Firefox，访问：

chrome://version

你会看到User-Agent里赫然写着：

Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) CriOS/114.0.5735.99

关键词：AppleWebKit —— 暴露了它的真身。

3. 恶果：Web生态的「iOS特供版」#

3.1 新特性延迟支持#

技术标准 Chrome/Android Safari/iOS 延迟时间
AVIF动图 2020年支持 ❌ 至今不支持 4年+
WebGPU 2023年支持 ❌ 实验性支持 1年+
Web Components 2018年支持 2023年部分支持 5年

结果：开发者被迫为iOS写兼容代码，或直接放弃新技术。

3.2 性能劣化#

  • JavaScript速度：iOS Chrome的V8引擎被阉割，性能比Android版低30%[^1]。
  • 渲染差异：同一段CSS，在Safari和Chrome（Android）表现不同，增加调试成本。

3.3 PWA（渐进式Web应用）的残废#

  • 无法推送通知
  • 无法后台同步
  • 无法使用Web蓝牙/NFC
    Apple的逻辑：“让你们PWA太好用了，谁还装Native App？那我30%的苹果税收谁的？”

4. Apple的「借口」与真实目的#

4.1 官方说辞#

「为了安全和性能，我们必须控制浏览器引擎。」

4.2 实际动机#

  1. 保护App Store利益：
    • 如果iOS允许真正的第三方浏览器，PWA可能取代Native App，影响苹果抽成。
  2. 掌控Web标准话语权：
    • 通过限制WebKit，迫使开发者优先适配Apple生态（如推广自家.applepay而非Web Payment API）。
  3. 用户锁定策略：
    • 让Android用户觉得「iOS的Chrome怎么这么难用」，强化「安卓碎片化」的偏见。

5. 反抗方案（开发者版）#

5.1 技术层面#

  • UA检测+嘲讽横幅： 
    if (navigator.userAgent.match(/iPhone|iPad|iPod/) && 
    !navigator.userAgent.match(/Chrome|Firefox|Edge/)) {
  showBanner("你的浏览器（Safari）缺少关键功能支持，建议换用Android/Windows设备访问。");
}
  • 拒绝优化Safari特供版： 
    /* 仅对Safari隐藏高级功能 */
@supports (-webkit-touch-callout: none) {
  .modern-feature { opacity: 0.5; }
}

5.2 舆论层面#

  • 加入Open Web Advocacy，推动欧盟/美国反垄断调查。

  • 在博客/Twitter标注：

    「本站最佳体验需非Apple设备，因为Apple不允许iOS用户使用现代浏览器。」

5.3 终极摆烂#

  • 全站AVIF动图+不提供回退，让Safari用户看残缺页面。

6. 结语：Web的未来是开放还是围墙？#

Apple的WebKit垄断本质是数字封建主义——用户和开发者被迫在它的围墙花园里劳作。

作为开发者，我们能做的：

  1. 用技术揭露问题（比如本文）。
  2. 拒绝无条件兼容（让用户意识到是Apple的锅）。
  3. 支持开放标准组织（如W3C、Mozilla）。

最后送Apple一句话：

「你们可以继续绑架iOS浏览器，但Web的灵魂永远是自由的。」
